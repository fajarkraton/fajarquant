\section{Reproduction Commands}
\label{sec:reprod}

All results in the paper can be reproduced from a clean checkout of the
\texttt{fajarkraton/fajarquant} repository. GPU access (NVIDIA with
$\geq$16\,GB VRAM) and a HuggingFace account with model access are
required.

\begin{lstlisting}[language=bash,caption={Full reproduction pipeline}]
# Clone and set up environment
git clone https://github.com/fajarkraton/fajarquant
cd fajarquant
python3 -m venv .venv
source .venv/bin/activate
pip install torch torchvision --index-url \
  https://download.pytorch.org/whl/cu124
pip install transformers datasets numpy

# Step 1: Calibrate PCA rotations (per-layer + per-head)
# Gemma (MQA, 1 head -- per-layer = per-head)
python3 scripts/calibrate_fq_v2.py \
  --model google/gemma-4-E2B \
  --output data/calibration/fq_v2_gemma_4_e2b.npz

# Mistral (GQA, 8 heads -- per-head calibration)
python3 scripts/calibrate_fq_v2.py \
  --model mistralai/Mistral-7B-v0.1 --per-head \
  --output data/calibration/fq_v2_mistral_7b_v0.1_perhead.npz

# Qwen2 (GQA, 4 heads -- per-head calibration)
python3 scripts/calibrate_fq_v2.py \
  --model Qwen/Qwen2-7B --per-head \
  --output data/calibration/fq_v2_qwen2_7b_perhead.npz

# Step 2: Profile KV heads
for model in google/gemma-4-E2B mistralai/Mistral-7B-v0.1 \
             Qwen/Qwen2-7B; do
  python3 scripts/profile_kv_heads.py --model "$model"
done

# Step 3: Tune thresholds + generate strategies
python3 scripts/tune_thresholds.py \
  --model mistralai/Mistral-7B-v0.1 --bits 3,4 \
  --calibration data/calibration/fq_v2_mistral_7b_v0.1_perhead.npz
python3 scripts/tune_thresholds.py \
  --model Qwen/Qwen2-7B --bits 3,4 \
  --calibration data/calibration/fq_v2_qwen2_7b_perhead.npz

# Step 4: PPL-guided 2-bit strategy selection
for model in google/gemma-4-E2B mistralai/Mistral-7B-v0.1 \
             Qwen/Qwen2-7B; do
  python3 scripts/ppl_guided_select.py --model "$model"
done

# Step 5: Full perplexity evaluation (Table 7)
# Gemma (seq_len=2048)
for bits in 2 3 4; do
  python3 scripts/eval_perplexity_v3.py \
    --model google/gemma-4-E2B \
    --strategy data/calibration/strategy_gemma_4_e2b_${bits}bit*.json \
    --calibration data/calibration/fq_v2_gemma_4_e2b.npz \
    --bits $bits --seq-len 2048 --max-samples 30 \
    --output data/kv_cache/perplexity_v3.1_gemma_${bits}bit.json
done

# Mistral (seq_len=2048)
for bits in 2 3 4; do
  python3 scripts/eval_perplexity_v3.py \
    --model mistralai/Mistral-7B-v0.1 \
    --strategy data/calibration/strategy_mistral_7b_v0.1_${bits}bit*.json \
    --calibration data/calibration/fq_v2_mistral_7b_v0.1_perhead.npz \
    --bits $bits --seq-len 2048 --max-samples 30 \
    --output data/kv_cache/perplexity_v3.1_mistral_${bits}bit.json
done

# Qwen2 (seq_len=1024 due to OOM at 2048)
for bits in 2 3 4; do
  python3 scripts/eval_perplexity_v3.py \
    --model Qwen/Qwen2-7B \
    --strategy data/calibration/strategy_qwen2_7b_${bits}bit*.json \
    --calibration data/calibration/fq_v2_qwen2_7b_perhead.npz \
    --bits $bits --seq-len 1024 --max-samples 30 \
    --output data/kv_cache/perplexity_v3.1_qwen2_${bits}bit.json
done

# Step 6: Verify paper tables
python3 scripts/verify_paper_tables.py --strict
\end{lstlisting}

\textbf{Expected runtime:} Steps 1--4 require $\sim$4 GPU hours
(dominated by calibration). Step~5 requires $\sim$4 GPU hours
($\sim$30 minutes per model$\times$bit evaluation). Total: $\sim$8 GPU
hours on RTX~4090.

\section{Qwen2 Sequence Length Note}

Qwen2-7B evaluations use $L{=}1024$ instead of $L{=}2048$ due to GPU
memory constraints. At $L{=}2048$, the model triggers CUDA OOM on our
16~GB RTX~4090 Laptop GPU because Qwen2's 28-layer, 28-head architecture
requires more activation memory than Gemma (15 layers) or Mistral (32
layers but efficient GQA-8). The shorter sequence length means Qwen2
perplexity values are not directly comparable in absolute terms to
Gemma/Mistral, but the \emph{relative ordering} between methods
(FQ~v3.1 vs KIVI vs TQ) is valid within the Qwen2 column.
